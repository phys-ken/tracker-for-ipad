# tracker_for_ipad — 改善設計メモ

生徒一人ひとりが自分のiPad（横向き）で、動画から運動を解析するためのツール。
最優先は「初心者が迷わない」こと。土台は既存の `app.js` を磨く方針。
リファレンス（IPhO2023 解析アプリ / Tracker Online）は概念の参考とし、コードは現行を発展させる。

## 合意した方針

### 1. 中核インタラクション（最大の変更）
- **中央十字＋確定ボタン方式**。十字は常に画面中央固定。
- 1本指 = 映像パン、2本指 = ズーム。「確定」で現フレームに点打ち/上書き → ステップ幅ぶん自動コマ送り。
- **モード切替（微調整/トラック）は完全廃止**。過去の点の修正は「表の行タップ → そのコマへ → 十字を合わせて確定で上書き」で一貫。

### 2. 校正
- 原点・スケールも同じ「十字に合わせて確定」作法に統一。
- バグ修正: `handleCanvasClick` の裸 `calibration` → `appState.calibration`（原点/スケール設定が ReferenceError で落ちていた）。

### 3. 動画・FPS
- `requestVideoFrameCallback` で実FPSを自動測定＋手動上書き。シークも同APIで精密化し、正しい時刻 t を保証。

### 4. オートトラック
- 残すが「おまけ」に格下げ。折りたたみパネル、難しいツマミは隠す/自動化。

### 5. データ出口（アプリ内グラフ＋外部出力のハイブリッド）
- グラフ: 表・グラフ・動画の点が**連動ハイライト**。位置に加え**速度・加速度グラフ**（数値微分、FPS依存）。Tracker を参考。
- 出力: **TSVコピー** ＋ **xlsxダウンロード**（SheetJS を CDN で追加）。

### 6. レイアウト
- 横向き前提。動画を主役に大きく。画面下に「コマ戻し / 確定 / コマ送り」を常時固定（親指到達）。校正・グラフ・表は折りたたみ/引き出し。

### 7. 物体
- 2〜3物体に簡素化。数値入力でなく色付き「物体1/2/3」ボタン。

### 8. 保全・導線
- Undo（直前の点打ち/削除取消）＋ localStorage自動保存（リロード復帰、動画は再選択）。
- 手順ガイド常設（①動画 ②スケール ③原点 ④トラッキング ⑤出力）＋空スタート。
- 「サンプルで試す」ボタン: `fetch('sample.mp4')` → Blob → 読込。生徒のお試し用 兼 自動テスト用バックドア（アップロードダイアログを回避）。

### 9. 依存・配布
- フォント/アイコンは CDN 継続（Wi-Fi安定前提）。
- 開発は3ファイル（index.html / app.js / styles.css）、配布はホスティングURL（GitHub Pages 候補）。
- ローカルは `serve.py`（:8000, LAN公開, キャッシュ無効化）。
- ブラウザ実機テストは Claude の Chrome 拡張で実施。

## ビジュアル・アイデンティティ（"計器 / ストロボ"）
主題（動画運動解析）の世界＝ストロボ運動写真＋計測器。ランディングではなく"計器"として作る。

- **配色**: `--ink #0F1216`（暗室グラファイト）/ `--panel #171B21` / `--line #2B333D`（ヘアライン＝方眼）/
  `--text #E6EAEF` / `--muted #8A95A3` / **`--trace #FFB627`（シグナル・アンバー）** / `--cyan #5AA9E6`。
- **2色のロジック**: アンバー＝**操作とライブ計測**（確定ボタン・照準レティクル・現ステップ・FRAME/FPS読み出し・
  スライダ・再生グリフ）。シアン＝**計器の枠と校正**（ビューファインダのコーナーブラケット・節タイトルの目印・
  原点/スケール値）。大胆さは「アンバーの確定ボタン」一点に集中、周囲は静かに。
- **書体**: IBM Plex Sans JP（工学のための書体・日本語＋ラテン）＋ IBM Plex Mono（計器の数値読み出し）。
  データ（t/x/y・コマ・FPS）は全てモノスペースでメーター化。
- **Signature**: ビューファインダ。動画をL字コーナーブラケット＋薄い方眼の"計器スクリーン"に収め、
  照準レティクルをアンバーで発光。位置マーカーはストロボの残像のように尾を引く（減衰ドット）。
- **品質の床**: レスポンシブ（〜768pxまで）、`:focus-visible` 可視、`prefers-reduced-motion` 尊重、`color-scheme: dark`。
