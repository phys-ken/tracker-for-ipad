# 動画解析トラッカー — 操作マニュアル

動画から物体の運動（位置・速度・加速度）を測定するツールです。
iPad のブラウザ（Safari / Chrome、**横向き推奨**）で動きます。スマホでも使えます。

> **スクリーンショットについて**
> 本マニュアルは各画面を ASCII 図で示しています。実機の写真を入れたい場合は
> `manual/img/` に画像を置き、各章の「📷」の箇所から参照してください
> （例: `![スケール設定](img/03-scale.png)`）。

---

## 1. 全体の流れ

測定はこの順番で進みます。画面上部の手順ガイドが「いま行うステップ」を**アンバー色**で点灯します。

```
① 動画を読み込む → ② スケール設定 → ③ 原点設定 → ④ トラッキング → ⑤ データ出力
```

- ② スケールと ③ 原点は**任意**です（未設定なら px 単位で測れます）。
- 物理量（cm・cm/s・cm/s²）で出したいなら ②③ を先に済ませてください。

---

## 2. 画面の構成

横向きの基本レイアウトです。

📷 `img/01-start.png`（起動直後）

**色の意味**
- 🟡 アンバー＝「操作・いま測っているもの」（確定ボタン・中央の照準・現在ステップ・コマ/FPSの数値）
- 🔵 シアン＝「計器の枠・校正」（スクリーン四隅の角枠・原点/スケールの値）

## 3. ① 動画を読み込む

2通りあります。

- **[動画を選択]** … iPad内の動画ファイルを選ぶ（実験で撮った動画はこちら）。
- **[サンプルで試す]** … 同梱のサンプル動画で練習できる。

読み込むと**自動で1回だけプレビュー再生**され、**その間に動画の本当のFPSを測定**します。
再生が終わると自動的に先頭フレームへ戻ります。
途中で十分だと思ったら **[▶]（一時停止）** を押せば、その場でプレビューを終えられます。

> **なぜプレビュー？** 動画ごとに実際のFPS（1秒あたりのコマ数）は違います。
> 正しいFPSが分からないと、時刻・速度・加速度がすべてズレます。読込時に自動測定して防ぎます。

📷 `img/02-load.png`（読込→プレビュー中）

## 4. ② スケール設定（長さの基準）

画面内の「**長さが分かっているもの**」（ものさし、既知の間隔など）を使って、
ピクセルと実寸（cm）の対応を教えます。

1. 右パネルの **[スケール設定]** を押す（ボタンが青く点灯）。
2. 映像を**1本指でドラッグ**して、中央の**十字を基準の片端**に合わせる。
3. **[確定]** を押す（始点）。
4. 十字を**もう一方の端**に合わせて、もう一度 **[確定]**（終点）。
5. 「実際の長さ」を **cm** で入力 → OK。

設定後、右パネルに `スケール: 0.500 cm/px` のように表示されます。

📷 `img/03-scale.png`

---

## 5. ③ 原点設定（座標の基準点）

測定の (0, 0) にしたい場所を決めます（例: 斜面の下端、発射点）。

1. **[原点を設定]** を押す（青く点灯）。
2. 映像をドラッグして**十字を原点にしたい位置**へ。
3. **[確定]**。

以後、x は右が＋、y は**上が＋**で記録されます。

📷 `img/04-origin.png`

---

## 6. ④ トラッキング（位置を打つ）

ここが中心の作業です。**指で点を打つのではなく、中央の十字に対象を合わせて [確定] する**方式です
（指で隠れず、ぴったり正確に打てます）。

**操作**
| やりたいこと | 操作 |
|---|---|
| 映像を動かす（対象を十字へ） | **1本指でドラッグ** |
| 拡大・縮小 | **2本指でピンチ** |
| 拡大をリセット | スクリーン右上の ⛶ ボタン |
| 点を打つ | **[確定（点を打つ）]** |

**手順**
1. 追跡したい瞬間まで **[<] [>]** でコマを送る。
2. 映像をドラッグ＆ピンチして、**十字を対象の中心**に合わせる。
3. **[確定]** を押す → その位置に点が記録され、**自動で次のコマへ進みます**。
4. また十字を合わせて [確定] … を繰り返すだけ。

**便利な設定（右パネル「コントロール」）**
- **ステップ幅** … 1コマずつでなく「2コマおき」等に間引いて打てます。
- **追跡する物体 [物体1/2/3]** … 衝突や2物体の実験は、色を切り替えて別々に追えます。

**間違えたとき**
- 直前の点を取り消す … **[↶元に戻す]**。
- 過去の点を直す … 下の「**測定データ**」表の**その行をタップ**→そのコマへ移動→十字を合わせ直して **[確定]** で上書き。
- 1点だけ消す … 表の行の **[削除]**、または点を選んで **[選択ポイントを削除]**。

📷 `img/05-track.png`

> **自動保存**: 打ったデータは自動で保存され、ページを再読み込みしても残ります
> （動画だけは選び直してください）。

---

## 7. ⑤ グラフを見る

右パネルの「リアルタイムグラフ」は、点を打つたびに自動更新されます。
プルダウンで種類を選べます。

- **位置**: y–t（垂直）／x–t（水平）／y–x（軌道）
- **速度**: 速さ–t／vx–t／vy–t
- **加速度**: 加速度–t／ax–t／ay–t
  （速度・加速度は隣り合う点から自動計算＝数値微分）

**連動ハイライト**: グラフ・表・動画上の点は連動します。
グラフの点をタップすると、その点が**アンバーで強調**され、動画もそのコマへジャンプします。

📷 `img/06-graph.png`

---

## 8. ⑥ データを出力

「測定データ」パネルの **[保存]** を押すと、出力ダイアログが開きます。
位置に加え、**速度・加速度の列も自動で付きます**。

- **[TSVをコピー]** … クリップボードへ。Numbers / Google スプレッドシート / Excel に**そのまま貼り付け**できます。
- **[TSV保存]** … タブ区切りテキストとして保存。
- **[xlsx保存]** … Excel 形式で保存。

出力される列:

```
object_id, frame, t (s), x, y, vx, vy, v, ax, ay, a
```

（x など長さの単位は、スケール設定済みなら cm、未設定なら px。速度は /s、加速度は /s²。）

📷 `img/07-export.png`

## 10. 困ったとき

- **動画が映らない / 真っ暗** … iPad では一部の動画形式が再生できないことがあります。
  標準のカメラ（MP4/H.264）で撮った動画を使ってください。`[サンプルで試す]` で動作確認を。
- **時刻や速度が変** … FPS が違う可能性。「コントロール」の **FPS** 欄に正しい値を手入力できます
  （手入力すると測定済みの時刻も計算し直します）。表示の `FPS: 30*` の `*` は「自動測定前／手動値」の印です。
- **コマがわずかにズレる** … 端末によってコマ単位のシークに誤差が出ることがあります。
  拡大して十字を合わせ直し、[確定] で上書きしてください。
- **色で自動追跡したい** … 右の「**オートトラック（補助）**」を開く → トラックモードで一度色を拾った点を作ってから
  「自動追跡」。背景が紛らわしいと外すので、外れたら手動で直してください（補助機能です）。
- **データが消えた？** … 自動保存されています。同じブラウザで開き直し、動画を選び直せば続きから使えます。

---

## 11. 起動・配布

- **生徒**: 配布された URL を iPad のブラウザで開くだけ（インストール不要、動画はアップロードされません＝端末内で処理）。
- **先生 / 開発**: ローカルで動かす場合は同梱の `serve.py` を実行し、
  `http://localhost:8000`（同一LANの iPad からは `http://<PCのIP>:8000`）を開きます。
```
python serve.py
```
